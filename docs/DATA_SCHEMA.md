# JD 数据接入 Schema

interview-forge **不内置任何招聘平台爬虫**。它只消费「结构化的 JD 数据」，数据怎么来由你
决定——手动整理、内部系统导出、或你自己的私有抓取脚本。只要产出符合下面 Schema 的数据即可。

提供两条对等的接入通道，任选其一（也可并用，按 `dedup_key` 去重）：

| 通道 | 命令 | 数据格式 | 配置项 |
|------|------|----------|--------|
| **JSON（推荐）** | `forge jobs-import [文件\|目录]` | `.json` | `jobs.json` |
| **Excel** | `forge jobs-scan` | `.xlsx` | `jobs.folder` |

## JSON 格式

顶层是一个**数组**，或一个含 `"jobs"` 数组的对象：

```json
[
  { "company": "...", "role": "...", "salary": "...", "desc": "..." },
  { "company": "...", "role": "...", "salary": "...", "desc": "..." }
]
```

单条记录的字段（**全部可选**，缺啥就空着）：

| 字段 | 含义 | 示例 |
|------|------|------|
| `company` | 公司名 | `"示例科技"` |
| `role` | 职位名 | `"高级算法工程师"` |
| `salary` | 薪资**原始字符串**（自动解析成 min/max/月数/年包） | `"30-50K·15薪"`、`"$120k-$160k"`、`"25-40K"` |
| `experience` | 经验要求 | `"3-5年"`、`"Mid-Senior"` |
| `education` | 学历要求 | `"本科及以上"` |
| `desc` | 岗位描述/职责（**最重要**，匹配与备考都靠它） | `"负责……要求……"` |
| `location` | 工作地点 | `"远程 / 深圳"` |
| `size` | 公司规模 | `"500-2000人"` |
| `bonus` | 加分项 | `"有 vLLM 落地经验"` |
| `url` | 投递/原始链接 | `"https://..."` |
| `industry` | 所属行业 | `"人工智能"` |

> 也兼容中文键名（`公司`/`岗位`/`职位`/`薪资`/`岗位描述`…），映射规则见
> `src/pipelines/jobs.py` 的 `COL_MAP`。

### 薪资字符串解析

`salary` 写人类可读的原始串即可，`parse_salary()` 会尽力抽取：
- `"30-50K·15薪"` → `min=30, max=50, months=15, annual_max=750`
- `"25-40K"` → `min=25, max=40, months=12, annual_max=480`
- `"$120k-$160k"` → `min=120, max=160`（按 K 量级）

解析不出来不影响入库，只是该岗位缺少薪资排序依据。

### 去重

入库按 `company|role|salary` 生成 `dedup_key`，跨文件、跨通道、重复导入都只保留一条；
嵌入向量只对**新行**计算，增量导入是秒级的。

## Excel 格式

`.xlsx` 首行为表头，列名按 `COL_MAP` 识别（中文表头）。把多个 xlsx 丢进 `jobs.folder`
目录，`forge jobs-scan` 会全部扫描入库（自动跳过 Excel 打开时的 `~$` 锁文件）。

## 简历

`forge resume`（或 UI「岗位匹配」页导入）读 `jobs.resume` 指向的简历文件，支持
`.pdf` / `.html` / `.txt` / `.md`，由 LLM 抽取成结构化画像（年限/方向/技能/项目/亮点）
并向量化，作为匹配与出题的「真实证据库」。
